Technical Subsystems

Platform Capabilities

A structural look at the backend algorithms and analytics driving candidate evaluation scores.

1. ATS Resume Parser

The system implements pythonic PDF parsers to extract candidate headers, email tags, and education credentials. It checks spelling structures, document page bounds, and compares textual content against target job keywords.

AUDIT CHECKS INVOLVED
  • Keyword density calculations (Python, Git, Docker, REST APIs).
  • Structural layout flags (detecting illegal graphic layers).
  • Page volume limits (penalizing redundant pages for freshers).

2. Readiness Score Engine

Uses multi-criteria decision scoring. The algorithm aggregates a candidate's overall academic CGPA, active certifications, registered projects, and skills to output a percentage match grade against individual company benchmarks.

SCORE METRIC FORMULA
  • Academic weightage (based on engineering branch matching).
  • Project complexity weights (languages used, summaries).
  • Verification checks (certificates matching target company stacks).

3. Skill Gap & Learning Paths

Once a target company is chosen, the engine performs a set difference operations between the company requirements and candidate skills. It highlights missing elements and generates weekly learning guides dynamically.

ROADMAP MODULES
  • Highlights missing languages, frameworks, or certifications.
  • Weekly modular resource guides pulled from curriculum structures.
  • Action items for database, styling, or architecture tasks.

4. Interview Simulation

Simulates technical rounds. The system queries specific skill categories and generates targeted questions (Conceptual, Coding, or Debugging) to prepare candidates for verbal and logic reviews.

SIMULATION ENGINE LOGIC
  • Pre-loaded skill indices matching interview checklists.
  • Dynamic question generation covering OOPs, databases, and APIs.
  • Self-paced evaluation checks with developer documentation links.
